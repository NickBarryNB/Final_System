# 基于RK3568边缘计算的智慧环境监测系统

## 2025-12-2

- 小组成员：王文富、杨鹏成、熊鲲

- 开发周期：2周<u>（2025-12-02至2025-12-16）</u>

- 经过组会讨论，达成以下共识：

  1. 核心目标：

     **用户端：**

     - 实现Web（部署在云服务器）和QT（部署在rk3568）的实时视频流、基础页面数据显示和各类控制按钮
     - 远程控制6路以上设备（LED灯、风扇、舵机、蜂鸣器、温湿度传感器、红外传感器

     **边缘节点端：**

     - 实现QT移植、mjpg-streamer/gstreamer视频流、CGI控制、Boa服务器、摄像头外设驱动、网络外设驱动等

     **终端：**

     - 实现6路以上设备（LED灯、风扇、舵机、蜂鸣器、温湿度传感器、红外传感器等）的外设，zigbee外设，先裸机实现后期可迭代freeRTOS进行多任务调度

  2. 技术路线：

     - RK3568跑Buildroot + mjpg-streamer视频流 + Boa Web服务器 + CGI控制；单片机（STM32）负责所有传感器外设数据上发和接收下发指令并执行，使用json数据格式通过zigbee/wifi/mqtt实现通信；web用ai，qt上位机界面和web端界面尽量一致

     - 日志异步

  3. 风险与对策：

     - 时间紧，优先保证“网页能看视频 + 点按钮能亮灯 + rk3568文件系统移植”，其余功能按优先级逐步添加。
     - rk3568移植SQLite暂未能实现
     - 多个功能需要自行学习
